# test-skill

Codex skills and deterministic engine code for short-video production workflows.

## Included Skills

- `short-video-editor`: Create reproducible workflows and editable assets for 1-2 minute vertical Chinese knowledge, explainer, or news commentary videos from an oral/talking-head video plus script.

## Install

Clone this repository, then run:

```bash
./install.sh
```

The script installs the skill to:

```text
${CODEX_HOME:-$HOME/.codex}/skills/short-video-editor
```

If an older local copy exists, it is moved to a timestamped backup path first.

## Install With Codex Skill Installer

If using Codex's skill installer, install from:

```text
repo: ryo1103/test-skill
path: skills/short-video-editor
```

## Quick Start For New Users

Prerequisites:

- Python 3.11 or newer.
- `ffmpeg` and `ffprobe` available on `PATH`.
- Optional but recommended: API keys for stock providers such as `PEXELS_API_KEY`, `PIXABAY_API_KEY`, or `COVERR_API_KEY`. Without them, the engine can still try free/no-key sources, but S3 may block if it cannot materialize enough distinct B-roll.

Prepare a project folder with these files:

```text
my-video-project/
  oral.mp4
  script.txt
```

`oral.mp4` is the source talking-head/oral video. `script.txt` is the exact source script. Then run:

```bash
git clone https://github.com/ryo1103/test-skill.git
cd test-skill
./install.sh
python3 skills/short-video-editor/scripts/init_short_video_project.py /path/to/my-video-project
python3 skills/short-video-editor/scripts/run_pipeline.py --project-dir /path/to/my-video-project --strict --enable-asr
python3 skills/short-video-editor/scripts/validate_production_acceptance.py --project-dir /path/to/my-video-project
```

Successful production output is written to:

```text
/path/to/my-video-project/output/final.mp4
/path/to/my-video-project/output/edit_package/
/path/to/my-video-project/work/plan/production_acceptance_report.json
```

Only claim completion when `production_acceptance_report.json` has `status=PASS` and `can_claim_complete=true`.

## Usage

After installation, restart Codex. The `short-video-editor` skill should be treated as a contract and command router, not as the editing executor.

## Architecture

- Skill = contract + command router. It tells Codex which engine commands to run and what must not be bypassed.
- Engine = deterministic execution. `skills/short-video-editor/engine/short_video_engine/` owns intake, subtitles, visual planning, assets, base timeline, motion, text layout, probe render, final render, and edit package generation.
- Validators = completion authority. PASS is trusted only when generated by the engine validator with provenance and matching artifact hashes.

Codex or a human editing JSON files directly cannot turn a failed project into a completed project.

## Pipeline Order

The engine runs subtitle readability planning before visual planning:

- `S0_intake`: read script and oral video metadata.
- `S1_script_and_subtitles`: create exact source script units and audio/manual-timed subtitle cues.
- `S1_5_subtitle_layout_planning`: split long subtitle display beats for 80-84px, max-two-line layout without modifying S1 source cues.
- `S2_visual_plan`: compile shot roles using script units, subtitle timing, and subtitle layout beats.
- `S3_asset_sourcing`: materialize distinct relevant local B-roll videos.
- `S4_base_timeline`: render the base plate from talking head and B-roll only.
- `S5_motion_overlay`: render deterministic logic motion artifacts.
- `S6_text_layout`: render title/subtitle ASS/SRT and layout evidence from the S1.5 plan.
- `S7_process_validation`: create probe render and representative frames.
- `S8_final_render_and_validation`: render final MP4, QC frames, edit package, and production acceptance.

## Canonical Usage

Initialize a project:

```bash
python3 skills/short-video-editor/scripts/init_short_video_project.py <project>
```

Run the production pipeline:

```bash
python3 skills/short-video-editor/scripts/run_pipeline.py --project-dir <project> --strict
```

Draft preview mode is allowed only for preview/debug workflows:

```bash
python3 skills/short-video-editor/scripts/run_pipeline.py --project-dir <project> --no-strict --draft-ok
```

Enable local ASR timing when no trusted manual/forced-alignment timestamps exist:

```bash
python3 skills/short-video-editor/scripts/run_pipeline.py --project-dir <project> --strict --enable-asr
```

The ASR producer uses local `faster-whisper` when available and writes `work/plan/asr_word_timestamps.json`. If ASR cannot run or cannot align to the source script with enough confidence, S1 remains `DRAFT_ONLY`; the engine does not relabel guessed timing as audio-derived.

Important completion rules:

- `output/FINAL_REPORT.md` is generated only after production acceptance PASS.
- `DRAFT_ONLY_REPORT.md` cannot be treated as a final deliverable.
- `FINAL_BLOCKED.md` means one or more real artifacts are still missing, invalid, or not trusted.
- Do not use `--allow-fixtures` in production. It is for tests only.

## Troubleshooting

- `S3_asset_sourcing` fail: real, relevant, distinct, materialized local video B-roll is insufficient, or downloaded assets lack source/license/decode evidence.
- `S5_motion_overlay` fail: required logic motion is not a real overlay artifact, is static, is only ASS/drawtext, or lacks start/mid/end evidence.
- `S8_final_render_and_validation` fail: `final.mp4` stream duration, last-frame timestamp, QC frames, motion/title/text evidence, or edit package media layers are incomplete.

Use the validator output failure codes as the source of truth:

```bash
python3 skills/short-video-editor/scripts/validate_stage.py --project-dir <project> --stage S3_asset_sourcing
python3 skills/short-video-editor/scripts/validate_final.py --project-dir <project>
python3 skills/short-video-editor/scripts/validate_production_acceptance.py --project-dir <project>
```

## Notes

This repository does not include production project videos, downloaded B-roll, rendered outputs, caches, local API keys, or temporary project render artifacts. Test media is generated at test time with ffmpeg.
